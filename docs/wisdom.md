# Wisdom

> 跨 issue 验证有效的可复用模式。
>
> 记录"为什么这样做有效"，而不是"做了什么"（那是 progress.md 的职责）。

## 维护规则

1. 只在 Stage 5（Reflection）写入，禁止在其他 Stage 追加
2. 每条必须有至少一个 issue_id 作为来源验证
3. 只写"在 X 场景下，Y 有效"，禁止写泛泛的"建议"或"最好"
4. 反例字段（不适用时）为必填——没有边界的模式会被过度泛化
5. 只追加，不修改历史条目

## 记录模板

```markdown
## W-00X 标题

- 来源：<issue_id> / PR 链接
- 适用场景：
- 模式描述：
- 反例（不适用时）：
```
